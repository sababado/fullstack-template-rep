# Version Management

This project follows [Semantic Versioning](https://semver.org/) with a format of MAJOR.MINOR.PATCH.

## Version Update Rules

### MAJOR Version (X.0.0)
Increment for **significant new features** and **architectural changes** or **breaking API changes**.

### MINOR Version (x.X.0)
Increment for **backward-compatible additions** (new features, new components).

### PATCH Version (x.x.X)
Increment for **bug fixes and minor improvements**.

## How to Update Version

1. Update version field in package.json (root and relevant packages).
2. Update CHANGELOG.md with a summary of changes.
3. Commit with message: chore: bump version to X.Y.Z.
